# ⚔️ CREW CONFLICT SYSTEM

ระบบนี้ใช้จำลอง “ความขัดแย้งภายใน Crew”  
ไม่ใช่แค่ทะเลาะกัน แต่คือ **แรงผลักดันของเนื้อเรื่องทั้งหมด**

ในโลกนี้:
👉 Crew ไม่ได้แตกเพราะแพ้ศัตรู  
👉 แต่แตกเพราะ “คนในทีม”

---

# 🧭 CONFLICT PHILOSOPHY

- Conflict is Story Engine
- Love, Hate, Trust = Core Mechanics
- No conflict = No growth
- Conflict must be believable, not random

---

# 🔥 TYPES OF CONFLICT

## 1. Ideology Conflict (ความคิดต่าง)
เช่น:
- จะช่วยคนหรือไม่
- จะฆ่าศัตรูหรือปล่อย
- จะเชื่อใจใคร

ผล:
- ลด Cohesion
- เพิ่ม Narrative Depth

---

## 2. Role Conflict (หน้าที่ขัดกัน)
เช่น:
- Navigator vs Captain (เส้นทาง)
- Doctor vs Combat (เสี่ยงชีวิต)
- Cook vs Supply shortage

ผล:
- ลด Morale ชั่วคราว
- สร้าง Synergy ใหม่ได้ถ้าจบดี

---

## 3. Emotional Conflict (อารมณ์)
เช่น:
- หึง / แค้น / เสียใจ / โทษตัวเอง

ผล:
- กระทบ Trust Network โดยตรง
- อาจ Trigger Trauma Memory

---

## 4. Leadership Conflict (อำนาจ)
เช่น:
- ไม่เชื่อ Captain
- มีคนท้าทายตำแหน่ง

ผล:
- เสี่ยง Mutiny Event
- แตกทีมได้จริง

---

## 5. Moral Conflict (ศีลธรรม)
เช่น:
- ปล่อยโจรที่ฆ่าคน
- ทิ้งเมืองเพื่อหนีเอาตัวรอด

ผล:
- เปลี่ยน Bond ระยะยาว
- เปลี่ยน Reputation ของ Crew

---

# 📊 CONFLICT INTENSITY

ทุกความขัดแย้งมีระดับ:

- Level 1: Minor disagreement
- Level 2: Heated argument
- Level 3: Trust damage
- Level 4: Relationship break
- Level 5: Crew fracture risk

---

# 🧠 CONFLICT RESOLUTION PATHS

## 1. Talk Resolution
- ใช้ Dialogue
- ลด Conflict แบบปลอดภัย

## 2. Action Resolution
- ใช้ภารกิจ / การพิสูจน์ตัวเอง
- เปลี่ยน Conflict เป็น Story Arc

## 3. Authority Resolution
- Captain ตัดสินใจ
- อาจสร้างคนไม่พอใจ

## 4. Breakdown Resolution
- ความสัมพันธ์พัง
- เกิด “Split Arc”

---

# ⚠️ MUTINY SYSTEM

ถ้าเกิด:

- Trust Network ต่ำมาก
- Morale ต่ำ
- Leadership Conflict Level 4+

จะเกิด:

## MUTINY EVENT

ผลลัพธ์:
- Crew อาจแบ่งเป็น 2 ฝ่าย
- Captain สูญเสียอำนาจชั่วคราว
- เกิด “Internal Boss Fight”

---

# 🔄 POST-CONFLICT EVOLUTION

ทุกความขัดแย้งต้อง “เปลี่ยนอะไรบางอย่าง”

- เพิ่ม Bond หรือ ลด Bond
- เปลี่ยน Role perception
- เปิด Memory ใหม่
- สร้าง Character Growth

---

# 🧩 SIMULATION RULE

Game Master ต้อง:

- ไม่แก้ Conflict ให้จบง่าย
- ต้องปล่อยให้ “มีผลจริง”
- Conflict ต้องทิ้งร่องรอยใน Memory
- ทุก Conflict ต้องเปลี่ยน Crew state

# CONFLICT MEMORY LINK

ทุก Conflict ระดับ 3 ขึ้นไป
ต้องถูกบันทึกลง CREW_MEMORY


Record:

- Participants
- Cause
- Decision
- Result
- Relationship Change


Example:

Event:
Captain ตัดสินใจเสี่ยงช่วยเกาะ


Participants:
Captain / Crew Members


Result:
บางคนไม่เห็นด้วย


Impact:

- Trust เพิ่มกับบางคน
- Trust ลดกับบางคน
- เกิด Shared Memory ใหม่
