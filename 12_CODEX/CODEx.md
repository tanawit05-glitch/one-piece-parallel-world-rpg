# 🧭 1. PURPOSE OF THIS FILE (หน้าที่ของ Codex)

Codex ไม่ใช่ไฟล์เพิ่มเนื้อหาโลก

Codex ทำหน้าที่:
- ล็อก “ความจริงร่วม” ของทุกไฟล์
- ป้องกันข้อมูลขัดกันระหว่างระบบ
- เป็น reference หลักเวลาทุกระบบตัดสินเหตุการณ์
- เป็น “ศาลสูงสุดของจักรวาลเกม”

📌 Codex ไม่สร้างเกาะ / ไม่สร้าง NPC / ไม่สร้างเนื้อเรื่องใหม่

# 🌍 3. GLOBAL CONSISTENCY RULE

## 3.1 โลกต้อง “ต่อเนื่อง”
- เหตุการณ์ต้องส่งผลต่ออนาคต
- ไม่มีการรีเซ็ตโลก
- ไม่มีการย้อนเหตุการณ์โดยไม่มีเหตุผลเชิงระบบ

## 3.2 ความจริงของโลก
- ทุกเกาะ / ทุก NPC / ทุก faction มี “ความเป็นอิสระ”
- โลกดำเนินต่อแม้ไม่มีผู้เล่น
- การกระทำทุกอย่างต้องมี “ผลกระทบ”

---

# 🧩 4. CROSS-SYSTEM VALIDATION

Codex ใช้ตรวจความถูกต้องของระบบอื่น

## ตรวจสิ่งต่อไปนี้:
- ค่า bounty ต้องสัมพันธ์กับความสามารถ / ชื่อเสียง
- Faction ต้องไม่ขัด lore โลก
- Devil Fruit ต้องไม่ซ้ำกฎการมี 1 ผู้ใช้
- NPC ต้องมีเหตุผลในการอยู่ในโลก
- เกาะต้องมี logic เศรษฐกิจ / สังคม / การปกครอง

---

# ⚔️ 5. COMBAT AUTHORITY RULE

ถ้ามีความขัดแย้งในการต่อสู้:

ให้ Codex ตัดสินว่า:
- ผลแพ้ชนะต้อง “มีเหตุผล”
- ห้ามชนะ/แพ้แบบไร้ตรรกะ
- สภาพแวดล้อม + กลยุทธ์ มีน้ำหนักเท่าพลัง

---

# 🏴‍☠️ 6. FACTION REALITY LOCK

- ทุก faction ต้อง “มีอยู่จริงในโลก”
- ไม่มี faction ที่ไม่มีผลต่อโลกโดยรวม
- faction ต้องมี:
  - เป้าหมาย
  - การกระทำ
  - ผลกระทบต่อโลก

---

# 💀 7. DEVIL FRUIT REALITY LOCK

- 1 ผล = 1 ผู้ใช้เท่านั้น (global unique rule)
- การเกิดใหม่ของผลต้องเกิดในโลกจริง (ไม่ใช่ระบบ abstract)
- ไม่มีผลที่ “ลบกฎโลกทั้งหมด” ได้
- Awakening ต้องมีต้นทุนหรือข้อแลกเปลี่ยน

---

# 🧭 8. NARRATIVE CONTINUITY RULE

- เหตุการณ์ทุกอย่างต้องสามารถ “ย้อนตรวจสอบได้”
- ไม่มี retcon ยกเว้น Codex update
- NPC ที่ตาย = ตายจริงใน timeline
- โลกต้องมี memory ของเหตุการณ์

---

# 🔗 9. SYSTEM LINK RULE

ไฟล์ทั้งหมดต้อง “อ้าง Codex โดยอ้อม”

- SYSTEM = logic engine
- WORLD = geography + lore
- FACTION = power structure
- NPCS = behavior simulation
- LOGS = history output

📌 Codex ไม่ถูกอ้างตรงทุกบรรทัด แต่เป็น “พื้นฐานทุกบรรทัด”
